# Интерпретатор лямбда-исчисления

Проект выполнен в рамках практики по курсу **"Математическая логика и теория алгоритмов"** (ФПМИ МФТИ, Петров Владислав Б05-515).

## Синтаксис модели
Вычислитель поддерживает следующий синтаксис:

* **Переменные:** строчные латинские буквы (`x`, `y`, `z`).
* **Абстракция:** `\var.body` (`\x.x`).
* **Аппликация:** `(M N)`.
* **Левая ассоциативность аппликации:** Выражения вида E1 E2 E3 .. En интерпритируются, как (...((E1 E2) E3)...En)
* **Область действия лямбды:** Тело абстракции простирается максимально далеко вправо, пока не встретится закрывающая скобка, ограничивающая данную абстракцию. То есть \x.MN эквивалентно (\x.(MN))
* **Сокращение вложенных абстракций:** Последовательность лямбд вида \x.\y.\z.M может быть записана как \xyz.M
### Макросы
Для упрощения кода поддерживается определение макросов. 
* Имена макросов состоят из **заглавных букв и цифр** (например, `TRUE`, `ADD`, `N1`).
* Синтаксис определения: `ИМЯ = <терм>`.
* Макросы должны быть определены до их использования. Целевое выражение для вычисления всегда находится в конце файла.

**Пример файла с макросами:**
```text
TRUE = \x.\y.x
FALSE = \x.\y.y
NOT = \p.p FALSE TRUE
NOT TRUE
```
## Как запустить
Для работы используется `uv`.

1. Установите зависимости: `uv sync`
2. Запустите интерпретатор: `uv run main.py <путь_к_файлу> --steps <число>`

### Параметры командной строки

* `file`: Путь к файлу с кодом для вычислителя. 

* `--steps`: Лимит шагов редукции (по умолчанию 1000). При превышении считается бесконечным циклом. 

* `-v, --verbose`: Включает явное логирование каждого шага редукции.

## Примеры программ (examples/)
1. `identity.txt`: Тождественное отображение `\x.x`.
2. `church_sum.txt`: Сложение чисел Чёрча(2 + 3).
